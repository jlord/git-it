关卡内容：

用 pull 来和其他 collaborators 同步更新，确保程序是最新的版本。

从 Remote pull 更新

若是和別人一起合作，就会需要把文件保持在最新状态，以免修改到相同的文件造成版本冲突。因此，你就会需要时常 pull 下其他人对程序所作的修改。

步骤：Reporobot 都做了什么呢？

把 GitHub 上 'origin' remote 的更新 pull 到你的电脑上，来看 Reporobot 是否有在 'add-' branch 做了任何修改：

$ git pull <REMOTENAME> <BRANCHNAME>

如果没有任何修改，Git 会告诉你 'Already up-to-date'。若有修改的话，Git 会把那些修改 merge 到电脑里哦。

Reporobot 有修改吗？Git 会告诉你哪些地方有变化。可以打开修改过的文件看看，Reporobot 修改了什么。哇！Reporobot 真是个艺术家呀！

确认你完成了所有步骤：

git-it verify

前往下一个关卡：

git-it

提示

  • 检查 Git 状态
    • $ git status
  • 从远程分支 pull 更新回来
    • $ git pull
  • 在收取更新前先检查 remote 是否有变动
    • $ git fetch --dry-run
