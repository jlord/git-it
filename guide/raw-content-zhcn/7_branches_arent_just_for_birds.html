关卡内容：

在你的 fork 新增 branch。

Branches

Git repositories 用 branches 来隔离进度。当需要跟其他人一起进行项目时，在你完成负责的部份之前，经常需要利用 branch 来保护你对程序所做的修改。如此，你就可以让主要的 'master' branch 保持稳定，不被未完成的修改影响。等到你完成在 branch 上的修改，就可以把它 merge 回 'master' branch。

在 GitHub Guide 中我们有提供解释 branches 运作方式的示意图：guides.github.com/overviews/flow

GitHub Pages

GitHub 会自动发布你放在 'gh-pages' branch 里的静态文件，并架设一个网站。由于你 forked 的项目内容是一个网站，所以它的预设 branch 是 'gh-pages' 而不是 'master'。而你会发现这些网站，网址都长的像这样：

http://githubusername.github.io/repositoryname

步骤：新增 branch

当你新增 branch 的时候，Git 会拷贝目前 branch 上所有的文件到新的 branch。

在终端输入 git status 可以知道目前你在哪个 branch（应该会在 'gh-pages'）。

新增一个 branch 并命名为「add-<username>」，请用你的账号名称替换掉 'username'。例如「add-jlord」。Branches 的名字有分大小写，所以请确定输入的账号名称跟 GitHub 上显示的一模一样。

$ git branch <BRANCHNAME>

你拥有了一个全新、内容跟 'gh-pages' 一模一样的 branch！

就像终端去另一个文件夹的指令 cd 一样，请 checkout 到刚才新增的 branch：

$ git checkout <BRANCHNAME>

步骤：新增文件

接下来我们回到文本编辑器：

  • 新增一个文件并命名为 「add-<USERNAME>.txt」，请用你的账号名称替换 'username'，像这样，「add-jlord.txt」。
  • 在文件里写下你的 GitHub 账号名称。例如，我就会写 'jlord'。
  • 将文件存到在 Patchwork 里的 'contributors' 文件夹： Patchwork/contributors/add-yourusername.txt
  • 接着，把你的修改记录下来！

步骤：记录

按照以下的步骤，把刚才的修改用 Git 记录下来：

$ git status

$ git add <FILENAME>

$ git commit -m "<commit message>"

Push 刚才记录好的修改到 GitHub 上，你 forked 的 repository 里头：

$ git push origin <BRANCHNAME>

确认你完成了所有步骤：

git-it verify

前往下一个关卡

git-it

git-it verify 的时候没有通过？

File NOT in contributors folder

刚才新增的文件应该要放到 Patchwork repository 的 'contributors' 文件夹里。如果不小心放到別的地方，请打开 Finder 或是 Windows 的文件管理器将该文件移到 'contributors' 文件夹，然后可以用 git status 看你刚才移动文件之后所造成的结果。用以下的命令 Stage 并且 commit 全部的修改（加上 -A，会将新增文件跟删除文件的动作一起记录下来）：

$ git add -A

$ git commit -m "move file into contributors folder"

Branch name expected: _____

Branch 的名字应该要跟你的账号名称一模一样。用以下的命令修改 branch 的名字：

$ git branch -M <NEWBRANCHNAME>

完成以上的动作之后，再执行一次 git-it verify！

提示

  • 只用一个指令就新增并切换到新的 branch
  • $ git checkout -b <BRANCHNAME>
  • 新增 branch
  • $ git branch <BRANCHNAME>
  • 切换到另一个 branch
  • $ git checkout <BRANCHNAME>
  • 列出所有的 branches
  • $ git branch
  • 重新命名目前所在的 branch
  • $ git branch -m <NEWBRANCHNAME>
  • 看目前正在哪个 branch
  • $ git status
